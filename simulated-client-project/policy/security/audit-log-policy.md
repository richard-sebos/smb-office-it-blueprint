## 1. Purpose

To ensure that audit logs generated by infrastructure and business applications are collected, protected, and retained in accordance with internal security standards and compliance goals. These logs support incident response, troubleshooting, and performance analysis.

---

## 2. Scope

Applies to:
- Linux servers (application, infrastructure)
- Samba AD domain controllers
- CUPS and file servers
- Workstations (via syslog or audit forwarding)
- Ansible automation logs
- All `auditd`, `rsyslog`, `journalctl`, and application logs

---

## 3. Policy Statements

- All systems **must generate logs** related to security, authentication, access, and system events.
- Logs **must not be modified or deleted** outside of automated retention policies.
- Logs should be **time-synchronized** using NTP and **timestamped** using UTC.
- **Auditd must be enabled** and configured on all Linux servers.
- Log access is **restricted to authorized personnel** and **read-only** for review.

---

## 4. Log Types Covered

| Log Source         | Purpose                           | Tool/Path              |
|--------------------|-----------------------------------|------------------------|
| `auditd`           | Security and file access          | `/var/log/audit/`      |
| `rsyslog/syslog`   | System events                     | `/var/log/syslog`      |
| `samba`            | AD and file access                | `/var/log/samba/`      |
| `cups`             | Print job tracking                | `/var/log/cups/`       |
| `sshd`             | Remote access events              | `/var/log/auth.log`    |
| `ansible`          | Automation run history            | `/var/log/ansible/`    |
| Application Logs   | Custom apps (as required)         | App-defined locations  |

---

## 5. Retention Requirements

| Log Type          | Retention Period | Rotation Tool     |
|-------------------|------------------|-------------------|
| `auditd`          | 180 days         | `logrotate.d`     |
| `syslog`          | 90 days          | `logrotate.d`     |
| Samba & CUPS      | 90 days          | `logrotate.d`     |
| SSHD              | 180 days         | `logrotate.d`     |
| Ansible Runs      | 60 days          | Custom rotation   |

- Compressed archives must be stored under `/var/log/archive/`
- Retention is automatically enforced via cron and logrotate
- Backups of logs are included in weekly backup plans

---

## 6. Log Access & Protection

- Logs are **read-only** for non-admin users
- Only the following groups may access logs:
  - `GG-Security`
  - `GG-Audit`
- All access to logs is logged and reviewed
- Logs must be **excluded from tamper-prone user writable partitions**
- Logs must not be stored on removable devices

---

## 7. Review & Monitoring

| Review Task                         | Frequency | Reviewer            |
|-------------------------------------|-----------|---------------------|
| Auditd rule hits (`ausearch`)       | Weekly    | IT Security Analyst |
| Log space usage                     | Weekly    | Linux Admin         |
| Access to audit logs                | Monthly   | IT Security Analyst |
| Integrity check (via AIDE)          | Weekly    | Security + Auditor  |

---

## 8. Enforcement

Violations of this policy may result in:
- Immediate revocation of access
- Incident response initiation
- Notification to HR (if employee-related)
- Project-wide security review
